SDK Quick-Start

Open a PAYG contract, sign requests, and make authenticated RPC calls to any Arkeo provider.

What you need: An Arkeo wallet with ARKEO tokens, a provider to subscribe to, and Node.js (for JS) or Python 3 (for Python).

1 Open a PAYG Contract

A contract locks a deposit of ARKEO tokens with a provider. You spend the deposit as you make requests.

Use the subscribe page to open a contract through your Keplr wallet. It handles transaction signing automatically.

After opening a contract, note your contract ID. You will need it for every authenticated request.

You can also open a contract programmatically using the Arkeo REST API:

# Open a PAYG contract with a provider
# deposit = amount in uarkeo (1 ARKEO = 100000000 uarkeo)
curl -X POST https://rest-seed.arkeo.network/arkeo/contract \
  -H 'Content-Type: application/json' \
  -d '{
    "type_url": "/arkeo.arkeo.MsgOpenContract",
    "value": {
      "creator": "YOUR_ARKEO_ADDRESS",
      "provider": "PROVIDER_PUBKEY",
      "service": "arkeo-mainnet-fullnode",
      "contract_type": "PAY_AS_YOU_GO",
      "deposit": "10000000",
      "duration": 648000
    }
  }'

Use the subscribe page for a guided UI-based flow instead.

2 Sign Requests (arkauth header)

Every request to an Arkeo sentinel must include an arkauth parameter. It proves you own the contract and tracks your nonce to prevent replay attacks.

Arkauth Format

{contractId}:{spender_pubkey_bech32}:{nonce}:{signature_hex}

How to Build It

  1. Build preimage string: {contractId}:{pubkey}:{nonce}
  2. Wrap in ADR-036 StdSignDoc (Cosmos arbitrary signing standard)
  3. SHA-256 hash the canonical JSON of the StdSignDoc
  4. Sign with secp256k1 (low-S normalized)
  5. Hex-encode the signature
  6. Append as query param: ?arkauth=...

Install dependencies:

cd sdk
npm install

Sign a single request manually:

import * as secp from '@noble/secp256k1';
import { sha256 } from '@noble/hashes/sha256';
import { hmac } from '@noble/hashes/hmac';

// Required setup for synchronous signing
secp.etc.hmacSha256Sync = (k, ...msgs) => hmac(sha256, k, secp.etc.concatBytes(...msgs));

const CONTRACT_ID = '942';
const PRIVATE_KEY_HEX = 'your_hex_private_key';
const PUBKEY_BECH32 = 'arkeopub1addwnpepq...'; // your bech32 public key
const SIGNER_ADDR = 'arkeo1...';          // your arkeo address

function buildArkauth(contractId, pubkeyBech32, signerAddr, nonce, privKeyHex) {
  const preimage = `${contractId}:${pubkeyBech32}:${nonce}`;
  const signDoc = JSON.stringify({
    account_number: "0", chain_id: "",
    fee: { amount: [], gas: "0" }, memo: "",
    msgs: [{ type: "sign/MsgSignData", value: {
      data: btoa(preimage),
      signer: signerAddr
    }}],
    sequence: "0"
  });
  const hash = sha256(new TextEncoder().encode(signDoc));
  const privBytes = Uint8Array.from(
    privKeyHex.match(/.{2}/g).map(h => parseInt(h, 16))
  );
  const sig = secp.sign(hash, privBytes, { lowS: true });
  const sigHex = Array.from(sig.toCompactRawBytes(), b =>
    b.toString(16).padStart(2, '0')
  ).join('');
  return `${contractId}:${pubkeyBech32}:${nonce}:${sigHex}`;
}

// Use it in a fetch call:
const nonce = 1;
const arkauth = buildArkauth(CONTRACT_ID, PUBKEY_BECH32, SIGNER_ADDR, nonce, PRIVATE_KEY_HEX);
const url = `https://sentinel.example.com/arkeo-mainnet-fullnode/status?arkauth=${encodeURIComponent(arkauth)}`;
const resp = await fetch(url);
console.log(await resp.json());

from arkeo_client import ArkeoClient
import requests, json, base64, hashlib
from ecdsa import SigningKey, SECP256k1

CONTRACT_ID = '942'
PRIVATE_KEY_HEX = 'your_hex_private_key'
PUBKEY_BECH32 = 'arkeopub1addwnpepq...'
SIGNER_ADDR = 'arkeo1...'

def build_arkauth(contract_id, pubkey_bech32, signer_addr, nonce, privkey_hex):
    preimage = f"{contract_id}:{pubkey_bech32}:{nonce}"
    sign_doc = json.dumps({
        "account_number": "0", "chain_id": "",
        "fee": {"amount": [], "gas": "0"}, "memo": "",
        "msgs": [{"type": "sign/MsgSignData", "value": {
            "data": base64.b64encode(preimage.encode()).decode(),
            "signer": signer_addr
        }}],
        "sequence": "0"
    }, separators=(',', ':'))
    msg_hash = hashlib.sha256(sign_doc.encode()).digest()
    sk = SigningKey.from_string(bytes.fromhex(privkey_hex), curve=SECP256k1)
    sig = sk.sign_digest(msg_hash, sigencode=lambda r, s, order: (r, s))
    r, s = sig
    # Normalize low-S
    order = SECP256k1.order
    if s > order // 2: s = order - s
    sig_hex = r.to_bytes(32, 'big').hex() + s.to_bytes(32, 'big').hex()
    return f"{contract_id}:{pubkey_bech32}:{nonce}:{sig_hex}"

nonce = 1
arkauth = build_arkauth(CONTRACT_ID, PUBKEY_BECH32, SIGNER_ADDR, nonce, PRIVATE_KEY_HEX)
url = f"https://sentinel.example.com/arkeo-mainnet-fullnode/status?arkauth={arkauth}"
resp = requests.get(url)
print(resp.json())

ADR-036 StdSignDoc

{
  "account_number": "0",
  "chain_id": "",
  "fee": { "amount": [], "gas": "0" },
  "memo": "",
  "msgs": [{
    "type": "sign/MsgSignData",
    "value": {
      "data": "<base64(contractId:pubkey:nonce)>",
      "signer": "<bech32_address>"
    }
  }],
  "sequence": "0"
}

The data field is the base64-encoded preimage string. The document must be canonical JSON (no extra whitespace).

3 Make Authenticated RPC Calls

Use the ArkeoClient class in sdk/arkeo-client.js to handle signing automatically. It auto-increments the nonce on each call.

// sdk/arkeo-client.js is in the sdk/ directory
import { ArkeoClient } from './arkeo-client.js'; // Node.js

const client = new ArkeoClient({
  sentinelUrl: 'https://sentinel.example.com',
  contractId: 942,
  privateKey: 'your_hex_private_key', // or 24-word mnemonic
  service: 'arkeo-mainnet-fullnode',
  startNonce: 1 // or fetch from chain (see Step 4)
});

// Get node status
const status = await client.rpcJson('/status');
console.log(status);

// Get block
const block = await client.rpcJson('/block?height=1');
console.log(block);

// Get latest block height
const info = await client.rpcJson('/abci_info');
console.log(info.result?.response?.last_block_height);

4 Nonce Handling

Nonces must strictly increment. If you restart your app, query the chain for the current nonce to avoid "bad nonce" errors.

Never reuse a nonce. Sentinels track nonces per-contract and will reject duplicate or out-of-order requests. 
// Fetch current nonce from chain before starting
async function getStartNonce(contractId) {
  const resp = await fetch(
    `https://rest-seed.arkeo.network/arkeo/contract/${contractId}`
  );
  if (!resp.ok) return 1;
  const data = await resp.json();
  return parseInt(data.contract?.nonce || '0') + 1;
}

const startNonce = await getStartNonce(942);
const client = new ArkeoClient({
  sentinelUrl: 'https://sentinel.example.com',
  contractId: 942,
  privateKey: 'your_private_key',
  service: 'arkeo-mainnet-fullnode',
  startNonce   // resume from correct nonce
});

// Persist nonce across restarts:
// const saved = JSON.parse(fs.readFileSync('nonce.json'));

// ... after each batch: fs.writeFileSync('nonce.json', JSON.stringify({nonce: client.getNonce()}));
import requests, json, os

def get_start_nonce(contract_id):
    resp = requests.get(
        f"https://rest-seed.arkeo.network/arkeo/contract/{contract_id}"
    )
    if not resp.ok: return 1
    data = resp.json()
    nonce = int(data.get('contract', {}).get('nonce', 0))
    return nonce + 1

# Or load from local state file:
NONCE_FILE = 'nonce_state.json'
def load_nonce():
    if os.path.exists(NONCE_FILE):
        return json.load(open(NONCE_FILE))['nonce']
    return get_start_nonce(942)

def save_nonce(nonce):
    json.dump({'nonce': nonce}, open(NONCE_FILE, 'w'))

start = load_nonce()
client = ArkeoClient(..., start_nonce=start)
# After requests: save_nonce(client.get_nonce())

Troubleshooting

"bad nonce" error

Your nonce is behind what the sentinel expects. Fetch the current nonce from the chain (see Step 4) and set your client to chainNonce + 1.

"invalid signature" error

Check that your private key matches the address that opened the contract. Verify the contract ID is correct. Ensure you are signing the preimage in the exact format: contractId:pubkeyBech32:nonce.

"unauthorized" or HTTP 401

Your contract may be expired or the deposit is fully consumed. Check your contract balance on the provider directory or by querying the REST API.

Connection refused

Verify the sentinel URL is correct and the service name matches your contract. Check the provider directory for the correct sentinel URL.

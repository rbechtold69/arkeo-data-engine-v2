Submit Your Frontend

The Arkeo Marketplace is open to anyone. Build a frontend that routes through Arkeo providers and get listed.

Permissionless Listing

Arkeo does not require permission to launch a frontend. Build your app, route it through Arkeo providers, and submit a GitHub issue to get listed in the marketplace. The community reviews submissions to keep quality high.

Requirements

🔓

Open Source

Your code must be publicly available on GitHub, GitLab, or another public repository. Users should be able to audit what they're running.

🔗

Routes Through Arkeo

All blockchain data requests must go through Arkeo providers using valid PAYG or subscription contracts. No bypassing the protocol.

🏗️

Legitimate Fork

If forked from another project, it must be a real improvement or specialization, not a copy with minor cosmetic changes.

📦

Static or Containerized

Must be deployable as a static site (GitHub Pages, Cloudflare Pages, etc.) or as a Docker container. No closed-source binaries.

🚫

No Malicious Code

No hidden trackers, no key loggers, no unauthorized fund transfers. The community will review your source before listing.

🤝

Community Responsive

Maintainers should be reachable on Discord or GitHub. Abandoned projects may be removed from the listing after community review.

How to Submit

1

Build and deploy your frontend

Make sure your app is live and publicly accessible. It should be using Arkeo providers for at least one core data function.

2

Open a GitHub Issue

Use the form below to generate a pre-filled issue on the arkeo-data-engine-v2 repository. Include your app name, URL, description, and repo link.

3

Join Discord and introduce yourself

Post in the #frontends channel on Arkeo Discord with a link to your GitHub issue. Community members will review and may ask questions.

4

Get listed

Once reviewed, a maintainer will add your frontend to the marketplace index. Listings are updated periodically.
